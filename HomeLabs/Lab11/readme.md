# 🌐 Лабораторная работа №11 — VPN. GRE. DMVPN

## 🎯 Цель работы
Настроить **GRE** между офисами Москва и С.-Петербург.  
Настроить **DMVPN** между Москва и Чокурдах, Лабытнанги.

---

## 📋 Задание

| № | Задача | Участники |
|---|--------|-----------|
| 1 | **GRE** Москва ↔ С.-Петербург | R15 ↔ R18 |
| 2 | **DMVPN** Москва (HUB) → Чокурдах (Spoke) | R15 ↔ R28 |
| 3 | **DMVPN** Москва (HUB) → Лабытнанги (Spoke) | R15 ↔ R27 |
| 4 | IP связность между всеми офисами | Все устройства |

---

## 📊 Топология сети

[Текстовая схема DMVPN/GRE](images/dmvpn_topologi.txt)

### Автономные системы и роли

| Устройство | Роль | AS |
|------------|------|-----|
| **R15** (Москва) | HUB (DMVPN) + GRE | 1001 |
| **R18** (С.-Петербург) | GRE Spoke | 2042 |
| **R28** (Чокурдах) | DMVPN Spoke | 2042 |
| **R27** (Лабытнанги) | DMVPN Spoke | - |

## 🔧 План настройки

### 1. GRE между Москвой и С.-Петербургом

| Параметр | R15 (Москва) | R18 (С.-Петербург) |
|----------|--------------|---------------------|
| Интерфейс | Tunnel0 | Tunnel0 |
| IP-адрес | 172.16.0.1/30 | 172.16.0.2/30 |
| Tunnel Source | Ethernet0/2 | Ethernet0/2 |
| Tunnel Destination | 100.0.0.10 | 100.77.0.6 |
| MTU | 1400 | 1400 |
| TCP MSS | 1360 | 1360 |

### 2. DMVPN между Москвой (HUB) и Чокурдахом, Лабытнанги

| Параметр | R15 (HUB) | R28 (Spoke) | R27 (Spoke) |
|----------|-----------|-------------|-------------|
| Интерфейс | Tunnel100 | Tunnel100 | Tunnel100 |
| IP-адрес | 172.16.1.254/24 | 172.16.1.1/24 | 172.16.1.2/24 |
| NHRP NHS | - | 172.16.1.254 | 172.16.1.254 |
| NBMA адрес | - | 100.77.0.6 | 100.77.0.6 |
| OSPF priority | 255 | 0 | 0 |
